# Agent AI Plan

This document defines behavior for fleets and player-like agents. The system
must be expandable, data-driven, and compatible with delayed information.

## Goal

Agents are ships or fleets making decisions under the same world rules as the
player:

- traders maximize profit;
- couriers deliver contracts and information;
- military fleets patrol, escort, raid, blockade, attack, and defend;
- scouts reveal systems and send reports;
- explorers seek unknown/high-value systems;
- adventurers behave like small player factions.

## Current State

`Agent` currently has:

```text
type
ship
currentStar
homeStar
destStar
money
cargo
targetFaction
missionCooldown
playerControlled
lastAction
```

`Game::updateAgents()` currently branches by type. This is acceptable for the
prototype but should evolve into data-driven role behavior.

## Agent Role Definition

Use a small role definition table:

```cpp
struct AgentRoleDef {
    std::string id;
    double tradeWeight;
    double courierWeight;
    double scoutWeight;
    double exploreWeight;
    double patrolWeight;
    double raidWeight;
    double colonizeWeight;
    double riskTolerance;
    double fuelReserveBias;
    double combatAvoidance;
};
```

An agent has one primary role but may score multiple actions.

## Decision Tick

No per-frame full-world scans.

```text
if missionCooldown > 0:
    wait
else if enRoute:
    movement system handles travel
else:
    evaluate limited candidate actions
    choose best score
    set order
```

Candidate limits:

- nearest known systems by cached spatial list;
- known market snapshots;
- local contracts;
- faction orders assigned to this agent;
- detected local threats;
- random exploration samples.

## Trader AI

Trader score:

```text
score =
    expectedProfit
  - fuelCost
  - routeRisk
  - staleInfoPenalty
  - capitalTieUpCost
```

Trader constraints:

- must reserve enough fuel for route or refuel stop;
- should not use live remote market truth unless debug/NPC omniscience is
  explicitly enabled;
- should avoid war zones unless riskTolerance is high.

## Courier AI

Couriers deliver cargo, settlement packets, faction orders, and contracts.

Courier score:

```text
score =
    reward
  + urgencyBonus
  + relationBenefit
  - fuelCost
  - routeRisk
  - delayPenalty
```

Courier tasks can be generated by `quests.md` and `events.md`.

## Scout AI

Scouts visit unknown or stale systems and emit reports.

Scout behavior:

1. Pick target system with high uncertainty or strategic value.
2. Fly there.
3. Observe owner, market summary, local fleets, hazards.
4. Create signal packets back to nearest owned/ allied relay star.
5. Continue outward or return to refuel.

Vision signal:

```text
sendTime = now
arrivalTime = now + dist3d(scoutPosition, relayStar)
payload = ScoutReport(...)
```

This is the core mechanic for faction knowledge.

## Explorer AI

Explorer differs from scout:

- scouts serve faction intelligence;
- explorers seek opportunity for themselves.

Explorer scoring:

```text
score =
    unknownValue
  + expectedResourceValue
  + contractDiscoveryValue
  - fuelCost
  - isolationRisk
```

Explorer-like agents can found tiny factions if they become rich.

## Military AI

Military role actions:

- patrol controlled systems;
- escort traders/couriers;
- intercept hostile fleets;
- raid enemy trade;
- blockade systems;
- attack colonies;
- retreat/refuel/repair.

Military score:

```text
score =
    strategicValue
  + factionOrderPriority
  + threatResponse
  - travelFuelCost
  - expectedCombatLoss
  - staleInfoPenalty
```

Local combat should be handled by `combat.md`.

## Player-Like Adventurers

Add about 100 small agents/factions that behave like the player:

- one ship;
- own money;
- no starting territory;
- can trade, take contracts, scout, pirate, flee, upgrade, found colony;
- may become notable faction if successful.

They use the same role scorer with personality weights.

```cpp
struct AdventurerProfile {
    double tradeBias;
    double questBias;
    double piracyBias;
    double colonyBias;
    double upgradeBias;
    double riskTolerance;
};
```

## Memory

Agents need local memory:

```text
knownMarkets
knownOwners
knownThreats
knownContracts
lastVisitedStar
homeRelayStar
```

Do not let normal agents read full simulation truth unless they are acting on
local observation or received signal payloads.

## Implementation Steps

1. Add role definition table.
2. Split current type branches into role scorer functions.
3. Add candidate caches for nearby/known systems.
4. Make player auto-trade use same trader scorer with player knowledge.
5. Add scout report signal generation.
6. Add courier contract acceptance.
7. Add military patrol and escort orders.
8. Add adventurer agents with personal profiles.
